## 04:54 — Critical Audio Investigation: Aliasing on Snare (Fixed)

**Problem:** Aliasing-like distortion, metallic coloration, zipper noise on snare drum.

**Root Causes Found & Fixed:**

1. **Bass filter Q=0.7 → 0.1** — Resonant lowpass ringing at 800 Hz was mixing with snare noise, creating intermodulation/metallic coloration. Reduced Q to 0.1 for clean rolloff.

2. **Missing anti-aliasing filters on drum voices** — Sample-based drum Players connected directly to Destination with no lowpass filtering. At 44.1k→48k sample-rate conversion boundaries (browser SRC), ultrasonic content aliases into audible range. Added 4th-order (rolloff: -24) 16 kHz lowpass filter per voice: `Player → Gain → AntiAliasFilter → Destination`.

3. **Zipper noise from instant gain changes** — `p.volume.value = linearToDb(...)` wrote dB directly to the Player's volume Param. Since `Param<"decibels">` triggers immediate AudioParam changes, each snare hit caused an abrupt gain step that sounds like aliasing on noise-rich transients. Replaced with a dedicated Gain node per voice, ramped via `linearRampToValueAtTime` over 5 ms.

4. **Bass volume also had no ramp** — `bass.volume.value = linearToDb(volume)` was instant. Changed to smooth dB write via Tone.getDraw().schedule for same reason.

**Architecture change:**
```
Before: Player.toDestination()
After:  Player → Tone.Gain (ramped) → Tone.Filter (16kHz LPF, -24dB) → Destination
```

**Files changed:**
- `src/audio.ts` — complete voice chain refactor

**Key decisions:**
- Voice chains (Gain+Filter) are created per-kit-load and disposed on kit swap
- `createAntiAliasFilter()` and `createVoiceChain()` factory pattern — clean init/cleanup
- Bass synth Q kept at 0.1 for clean low-end (empirical choice)
- Filter frequency 16 kHz (below 22.05k Nyquist for 44.1k, with margin)
